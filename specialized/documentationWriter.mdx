# Documentation Writer Agent

The Documentation Writer agent automatically generates technical documentation from source code, including API references, README files, and user guides.

## Capabilities

### API Documentation

Generates comprehensive API documentation using [[functions/parseCode]] and [[functions/generateDocs]]:

- Parse function signatures and type annotations
- Extract JSDoc, docstring, and XML comments
- Document parameters, return types, exceptions
- Generate usage examples
- Include authentication requirements
- Document rate limits and constraints

### README Generation

Creates professional README files with:

- Project overview and features
- Installation instructions
- Quick start examples
- Configuration options
- Contributing guidelines
- License information

### User Guides

Writes end-user documentation:

- Step-by-step tutorials
- Common use cases and examples
- Troubleshooting guides
- Best practices
- FAQ sections

### Code Comments

Enhances code with inline documentation:

- Explain complex algorithms
- Document design decisions
- Add usage examples
- Note limitations and edge cases
- Link to related documentation

## Integration with Workflows

This agent is part of [[workflows/releaseVersion]], automatically generating documentation before each release.

## Supported Languages

- JavaScript/TypeScript (JSDoc)
- Python (docstrings)
- Java (Javadoc)
- C# (XML comments)
- Go (godoc)
- Rust (rustdoc)
- Ruby (RDoc)

## Example API Documentation

```markdown
# Task Management API

## Overview

RESTful API for task management with authentication, collaboration, and real-time updates.

**Base URL:** `https://api.example.com/v1`

## Authentication

All API requests require authentication using Bearer tokens:

```http
Authorization: Bearer YOUR_API_KEY
```

Get your API key from the [dashboard](https://app.example.com/settings/api).

## Endpoints

### List Tasks

```http
GET /tasks
```

Retrieves a list of tasks for the authenticated user.

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| page | integer | 1 | Page number |
| per_page | integer | 20 | Items per page (max 100) |
| status | string | all | Filter by status: pending, completed, all |
| sort | string | created_at | Sort field: created_at, due_date, title |
| order | string | desc | Sort order: asc, desc |

**Response:**

```json
{
  "tasks": [
    {
      "id": "task_123",
      "title": "Complete project",
      "description": "Finish the documentation",
      "status": "pending",
      "due_date": "2025-12-31T23:59:59Z",
      "created_at": "2025-10-04T10:00:00Z",
      "updated_at": "2025-10-04T10:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 45,
    "pages": 3
  }
}
```

**Example:**

```bash
curl -X GET "https://api.example.com/v1/tasks?status=pending&sort=due_date" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Create Task

```http
POST /tasks
```

Creates a new task.

**Request Body:**

```json
{
  "title": "Task title",
  "description": "Task description",
  "due_date": "2025-12-31T23:59:59Z",
  "assignee_id": "user_456",
  "tags": ["important", "urgent"]
}
```

**Response:**

```json
{
  "id": "task_789",
  "title": "Task title",
  "status": "pending",
  "created_at": "2025-10-04T10:30:00Z"
}
```

**Example:**

```javascript
const response = await fetch('https://api.example.com/v1/tasks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title: 'Complete documentation',
    description: 'Write API docs',
    due_date: '2025-12-31T23:59:59Z'
  })
})

const task = await response.json()
console.log('Created task:', task.id)
```

## Error Handling

The API uses standard HTTP status codes:

| Code | Description |
|------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing API key |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found - Resource doesn't exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |

**Error Response:**

```json
{
  "error": {
    "code": "invalid_parameter",
    "message": "The 'due_date' field must be a valid ISO 8601 timestamp",
    "field": "due_date"
  }
}
```

## Rate Limits

- **Free tier:** 100 requests per minute
- **Pro tier:** 1000 requests per minute
- **Enterprise:** Custom limits

Rate limit headers are included in all responses:

```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1609459200
```
